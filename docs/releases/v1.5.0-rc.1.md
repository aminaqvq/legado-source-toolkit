# v1.5.0-rc.1 — Single Source Lab

**Release Candidate 1** — 2026-06-13

> This release introduces the first Release Candidate of Single Source Lab, a staged validation workflow for Legado book sources. It is designed to automate common source validation steps, not to fully replace the Legado Android runtime.

## Highlights

- **Real chain validation**: search → bookInfo → toc → content with actual URL linking between stages
- **Scoped rule execution**: item-scope sub-rule extraction for search results and TOC chapters
- **JSON search support**: JSONPath bookList with shorthand field names
- **Structured debugging**: per-stage network traces, rule traces, and extraction results
- **Debug UI**: Web GUI "单源调试" page with search results table, TOC table, content preview

## Added

### Types

- `SearchItem` — structured search result with name, author, bookUrl, confidence
- `ChapterItem` — chapter with chapterName and chapterUrl
- `BookInfoResult` — book info with name, author, intro, coverUrl, tocUrl
- `ContentResult` — content with contentLength, contentPreview, isTooShort
- `NetworkTrace` — HTTP request trace (url, method, status, finalUrl, responseSize, error)
- `RuleTrace` — rule execution trace per rule
- `StageResult` — full stage result with extracted data, errors, suggestions
- `FailureReason` — machine-readable error classification enum

### Engine

- `resolveScopedPipeline()` in `executor.ts` — safely splits `selector@getter` rules without breaking `@@`, `@js:`, CSS attribute selectors
- `executeRuleOnScope()` — scoped rule execution for html-element, json-object, and text scope kinds
- `detectItemKind()` — dynamic detection of cheerio elements vs JSON objects vs plain text
- `resolveCandidateUrl()` in `url-resolver.ts` — unified URL resolution for absolute, relative, `//`, and domain-like candidates

### Verification

- `extractSearchItems()` — extracts SearchItem[] from search page using per-item scoped rules
- `extractChapterItems()` — extracts ChapterItem[] from TOC page using per-item scoped rules
- `extractBookInfo()` — extracts BookInfoResult with pipeline-aware tocUrl/coverUrl extraction
- `verifyAllRules()` — orchestrates full chain: search → bookUrl → bookInfo → tocUrl → toc → chapterUrl → content

### Network

- `safeFetch()` now supports POST method, request body, and custom headers
- Manual redirect loop with per-hop SSRF re-check (max 5 hops)
- `failureReason` field on `SafeFetchResult`: `ssrf_blocked` | `timeout` | `dns_failed` | `connection_error` | `network_error`
- `mapFetchFailure()` helper ensures all four stages report accurate errors

### Debug API

- `POST /api/debug` response enriched with `extracted`, `request`, `errors`, `suggestions` per stage

### Debug UI

- Search results table (name, author, bookUrl, confidence)
- TOC chapters table (chapterName, chapterUrl)
- Content preview with length and isTooShort indicator
- Network request collapsible panel (url, method, status, responseSize, bodyPreview)
- BookInfo detail view (name, author, tocUrl, coverUrl, intro)
- Errors and suggestions badges per stage

### Tests

- Full orchestrator chain test with mock fetch (4 stages, all RULE_VERIFIED, fetch call count and order)
- `book_url_missing` stop-on-failure test (1 fetch, 1 stage)
- `chapter_url_missing` stop-on-failure test (3 fetches, 3 stages)
- `content_too_short` direct verifyRuleContent test
- JSON search source test in verifyRuleSearch
- Scoped pipeline tests: `.name@href`, `a@href`, `@href`, `@text`, `@src`
- JSONPath item scope tests with shorthand field names
- `resolveCandidateUrl` tests (absolute, relative, `//`, domain-like, invalid)

## Fixed

- Removed fake `extractFirstBookUrl` fallback to `bookSourceUrl`
- Removed empty `extractFirstChapterUrl` that always returned `''`
- Fixed `content_too_short` unreachable dead branch (`if (len < 20) { if (len >= 20) { ... } }`)
- Prevented false `RULE_VERIFIED` when no bookUrl or chapterUrl extracted
- Fixed URL base resolution to use `fetchResult.finalUrl` instead of hardcoded `bookSourceUrl`
- Fixed `getElementAttribute` case sensitivity for `@href`, `@src`, `@text`
- Fixed `SCOPE_GETTERS` case normalization (`@outerHtml` → `@outerhtml`)
- Fixed `extractBookInfo` to support pipeline rules like `.read-btn@href`

## Security

- Manual redirect handling replaces `redirect: 'follow'` — every hop gets SSRF re-check
- Per-hop SSRF validation via `isSafeURL()` DNS resolution + private IP check
- Dangerous header filtering: Host, Cookie, Authorization, Proxy-Authorization, X-Forwarded-For, X-Real-IP
- Response body size limit (256KB)
- `mapFetchFailure()` maps fetch errors to distinct failure reasons instead of generic `empty_response`
- `webView:true` and `java.ajax` detected and reported as unsupported, not executed

## Known Limitations

| Feature | Status |
|---------|--------|
| WebView rendering | Not supported — requires Browser Runner |
| `java.ajax` | Blocked by default in safe sandbox |
| `java.getCookie` | Not supported |
| `Packages.*` imports | Not supported |
| Rhino engine | Not fully simulated |
| Login session management | Not implemented |
| Cloudflare / CAPTCHA bypass | Detected, not bypassed |
| JS-rendered pages | Only server-rendered HTML and JSON APIs |
